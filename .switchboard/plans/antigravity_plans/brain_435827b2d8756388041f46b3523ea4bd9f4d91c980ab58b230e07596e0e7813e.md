# ระบบ Admin และการตั้งค่า Dark Mode เริ่มต้น

เพิ่มความปลอดภัยในการจัดการข้อมูลด้วยระบบ Admin Mode และปรับแต่งประสบการณ์การใช้งานเริ่มต้นให้น่าประทับใจด้วย Dark Mode

##Proposed Changes

### Authentication & Theme Setup

- ปรับปรุง Logic ใน `useEffect` ของ Theme Handling ให้ตั้งค่าเป็น `dark` ตั้งแต่เริ่มต้น
- เพิ่มตัวแปรสถานะ `isAdminAuthenticated` เพื่อควบคุมสิทธิ์ในระดับแอปพลิเคชัน
- เพิ่ม UI สำหรับการกรอกรหัสผ่าน Admin (รหัส: `31020177`) เมื่อพยายามใช้งานฟังก์ชันที่ต้องการสิทธิ์

### Authorization Logic (Admin Enforcement)
#### [MODIFY] [grading-system.tsx](file:///f:/webdev/รวมโปรเจคของโรงเรียน/lhb.pages.dev/lhb-grades/components/grading-system/grading-system.tsx)
- **จัดการนักเรียน**: การเพิ่ม, ลบ และการนำเข้า (Import) จะต้องได้รับการยืนยันสิทธิ์ Admin ก่อน
- **จัดการรายวิชา**: การแก้ไขชื่อ, หน่วยกิต และการนำเข้าวิชาใหม่ จะต้องใช้สิทธิ์ Admin เช่นกัน
- ปรับปรุง UI ให้ซ่อนหรือปิดการใช้งาน (Disable) ปุ่มที่ต้องใช้สิทธิ์ หากยังไม่ได้ล็อกอินเป็น Admin

## Verification Plan

### Manual Verification
- [1] **Dark Mode Default**: ลบ LocalStorage แล้วโหลดหน้าเว็บใหม่ ระบบต้องแสดงผลเป็นโหมดมืดทันที
- [2] **Admin Authentication**:
    - ทดสอบคลิกปุ่ม "เพิ่มรายวิชา" หรือ "นำเข้านักเรียน" ระบบต้องแสดง Modal ให้ใส่รหัสผ่าน
    - กรอกรหัส `31020177` ต้องสามารถเข้าใช้งานส่วนต่างๆ ได้
- [3] **Access Control**: ตรวจสอบว่าหลังจากเป็น Admin แล้ว ฟังก์ชันที่ต้องการสิทธิ์ (แก้ไข/ลบ/นำเข้า) ทั้งหมดสามารถใช้งานได้ปกติ
